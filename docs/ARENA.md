# ARENA — арена, маза, навмеш

Файлы:
- `src/arena/arena_view.gd` (460 строк) — пол, стены, спавн объектов.
- `src/arena/maze_generator.gd` — алгоритм каменного лабиринта.

---

## 1. Размер и пол

**Арена 36×24** (X×Z). Пол — тайлы `KayKit Floor_Dirt.obj`, материал `#3D2E1F` (тёплый коричневый), случайный поворот каждого тайла для разнообразия.

**Единая коллизия пола** — один `StaticBody3D` + `BoxShape3D` на всю арену (не покадровая на тайл) для производительности.

---

## 2. Стены

4 стены из `Primitive_Wall.obj` + `Primitive_Wall_Half.obj` для углов.

- Высота: `WALL_HEIGHT` (для коллизии).
- Толщина: `WALL_THICKNESS`.
- Коллизии — отдельные BoxShape вокруг арены, чуть выходят за пол на `half_thick`.

---

## 3. MazeGenerator — каменный лабиринт

Файл: `src/arena/maze_generator.gd`.

| Константа | Значение | Описание |
|---|---|---|
| `RIDGE_COUNT` | 1080 | Целевое число гряд (после фильтрации останется меньше) |
| `MIN_GAP` | 2 | Минимальный зазор между грядами в ячейках |
| `MAX_RIDGE_LENGTH` | 12 | Длиннейшая гряда |
| `SHORT_RIDGE_MAX_RATIO` | 0.15 | Доля коротких (1–2 ячейки) от общего числа |
| `SAFE_ZONE` | (центральная зона) | Гарантированно свободная область вокруг центра спавна игрока |

**Алгоритм:** перебор `RIDGE_COUNT` итераций — на каждой пытается разместить `Ridge` (горизонтальный или вертикальный сегмент длиной `randi_range(min_len, MAX_RIDGE_LENGTH)`). Гряда принимается если:
- Не выходит за границы (с учётом отступа 2).
- Не пересекает занятые ячейки (с учётом MIN_GAP).
- Если короткая — счётчик коротких не превышает `floori(RIDGE_COUNT * SHORT_RIDGE_MAX_RATIO)`.

**Меши:**
- Короткие гряды (≤2 ячеек): `Pallet_Small_Decorated`.
- Длинные: `Cube_Prototype_Large`.

## 4. NavMesh бейкинг

`NavigationRegion3D` баится с источниками:

1. **Невидимый `BoxMesh`-пол** как child NavigationRegion (плоский планар-источник). Без этого нав-меш получается с 0 полигонов.
2. **Копии коллайдеров камней** — добавляются как mesh-источники, чтобы бейкинг вырезал зоны вокруг камней (враги не могут пройти).

**Параметры:**
- `cell_size = 0.5`
- `agent_radius = 0.4`

После бейка все служебные мешки скрываются (`visible = false`).

**Связано:** в `enemy_base.gd` агенты используют `NavigationAgent3D` с `radius = 0.3` (меньше agent_radius, чтобы они проходили в проёмах).

## 5. Безопасный спавн объектов

`get_distributed_spawn_positions(count, center_pos, min_distance)`:
- Использует `NavigationServer3D.map_get_random_point` — точка гарантированно на навмеше (достижима врагами/игроком).
- Минимальная дистанция от `center_pos` (от игрока) и между точками.

Применяется для:
- Спавна врагов (5 шт).
- Респауна книг (после использования).

## 6. Декор пола

`_populate_floor_decor` — на свободных тайлах (не занятых грядами) ставит `Trapdoor.glb` со случайным шансом 0.1%.

- Без коллайдера, не блокирует ничего.
- `metallic = 0`, `roughness = 1` — без бликов.
- `Y = 0` — часть пола, **не закрывает индикаторы** (на Y=0.25).

---

## 7. Edge-кейсы

- ✅ **NavMesh с 0 полигонов** — решено: невидимый BoxMesh-пол как источник (без него нав-меш не строится).
- ✅ **Камни перекрывают спавн** — `SAFE_ZONE` в центре MazeGenerator.
- ⚠️ **Игрок застрял в зазоре между камнями** — `MIN_GAP = 2` обеспечивает минимум 2 тайла между грядами; для агента `radius=0.3` это достаточно.
- ❓ **Перегенерация на рестарт** — сейчас арена создаётся один раз. Рестарт через R сохраняет ту же геометрию. Для другой арены — `seed` для random или регенерация при переходе на новую комнату.
