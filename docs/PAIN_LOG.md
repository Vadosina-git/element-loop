# PAIN_LOG — журнал решённых проблем

Список багов, на которые мы уже наступили и нашли решение. Прочти в начале сложной задачи, если её тема похожа на одну из ниже — не повторяй ошибки.

Формат: **симптом** → **причина** → **решение**.

---

## Навигация и движение

- **NavMesh с 0 полигонов** → нет mesh-источника → невидимый `BoxMesh`-пол как child `NavigationRegion3D`.
- **Враги «замерзают» на месте при движении** → `NavigationAgent3D.get_next_path_position()` возвращает точку с другой высотой → выравнять `next_pos.y = global_position.y` перед расчётом direction.
- **Враги «вибрируют» (дёргаются туда-сюда)** → пересчёт направления каждый кадр конфликтует с физикой → кешировать direction раз в 0.3 сек (`_nav_update_timer`), плавно лерпить velocity между кадрами.
- **NavMesh fallback спам в логе** → у ranged/bomber `get_chase_position()` возвращает текущую позицию (стоят в кайт-зоне) → NavAgent сразу финиширует → раньше валилось в фолбэк с `push_warning`. **Решено:** если goal_pos ближе 0.3м к врагу, NavAgent не дёргается, `_cached_move_dir = ZERO`. Фолбэк-варн убран как нерелевантный.

## Рендер и UI

- **Эмодзи не рендерятся в браузере** (Godot Web export) → fallback-фонт неполный → использовать **PNG-иконки 160×160** из Apple Color Emoji (см. `assets/icons/`).
- **Пост-обработка перекрывает HUD** → `CanvasLayer` пост-эффектов был выше HUD по `layer` → ставить `layer = 0`, HUD на дефолтном (выше).
- **Outline делает игрока белым** → `next_pass` материал применяется ко всем surface через материал-shared → **клонировать surface materials** перед добавлением next_pass.
- **Декор пола перекрывает индикаторы** под ногами → `Trapdoor.glb` на Y=0, индикаторы тоже были на Y=0 → поднять индикаторы на **Y=0.25**.

## Управление

- **Игрок «шёл задом»** на KayKit моделях → KayKit-модели смотрят по `+Z`, Godot ожидает `−Z` → повернуть mesh на `rotation.y = PI`.
- **Клавиши не работают в браузере** на нелатинских раскладках (русская/греческая/иврит) → `event.keycode` возвращает локаль-зависимый код → использовать `event.physical_keycode` как фоллбек.

## Снаряды

- **`!is_inside_tree()` ошибка при спавне индикатора бомбы** → `set_as_top_level(true)` и `global_position` вызваны до `add_child` → переставить вызов после добавления в дерево.

## ImmediateMesh

- **`No vertices were added, surface can't be created`** → `surface_begin/end` без `surface_add_vertex` между ними → гейтить вызов: открывать surface только если **есть что рисовать** (см. `DEBUG_VISUAL.md`).
